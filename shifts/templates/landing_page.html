medip-shifts

Pensadas para ser consumidas como un sencillo API que devuelve un único valor en texto plano, las operaciones de consulta que se proporcionan son:

/current_week
Obtiene la persona asignada al turno durante la semana actual
/next_week
Obtiene la persona asignada al turno durante la próxima semana

Relacionadas con el estado de la sesión y para usar de forma interactiva existen:

/login
Inicia una sesión autenticada con un usuario
/logout
Cierra la sesión autenticada con un usuario
/change_password
Posibilita el cambio de contraseña

También de uso interactivo, las operaciones que modifican el estado de los turnos son:

/switch_shifts
Posibilita el intercambio de turnos entre dos fechas (necesita autenticación)
/reset
Regenera los turnos iniciales (necesita autenticación). ATENCIÓN: Esta operación es irreversible
